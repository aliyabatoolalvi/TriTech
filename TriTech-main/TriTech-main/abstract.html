Abstraction

Abstraction is one of the four fundamental pillars of object-oriented programming (OOP) in Java. It involves simplifying complex systems by modeling classes based on the essential properties and behaviors they possess. Abstraction allows you to focus on the relevant details of an object while hiding unnecessary complexity. There are two main ways to achieve abstraction in Java: abstract classes and interfaces.

1.Abstract Classes:

An abstract class is a class that cannot be instantiated and may contain abstract methods (methods without a body) that must be implemented by its concrete subclasses.
// Abstract class
abstract class Shape {
// Abstract method
public abstract void draw();
// Concrete method
public void displayArea() {
System.out.println("Area calculation is specific to each shape.");
} }
// Concrete subclass
class Circle extends Shape {
@Override
public void draw() {
System.out.println("Drawing a circle.");
} }
// Concrete subclass
class Rectangle extends Shape {
@Override
public void draw() {
System.out.println("Drawing a rectangle."); } }
In this example, the Shape class is an abstract class with an abstract method draw. Concrete subclasses (Circle and Rectangle) extend the abstract class and provide specific implementations for the draw method.

2. Interfaces:

An interface is a collection of abstract methods that define a contract for classes that implement the interface. In Java, a class can implement multiple interfaces.
// Interface
interface Shape {
// Abstract method
void draw();
// Default method with a body
default void displayArea() {
System.out.println("Area calculation is specific to each shape.");
} }
// Concrete class implementing the interface
class Circle implements Shape {
@Override
public void draw() {
System.out.println("Drawing a circle."); } }
// Concrete class implementing the interface
class Rectangle implements Shape {
@Override
public void draw() {
System.out.println("Drawing a rectangle."); } }
In this example, the Shape interface defines an abstract method draw and a default method displayArea. The Circle and Rectangle classes implement the Shape interface and provide specific implementations for the draw method.

Using Abstraction

public class Main {
public static void main(String[] args) {
Shape circle = new Circle();
circle.draw(); // Output: Drawing a circle
circle.displayArea(); // Output: Area calculation is specific to each shape
Shape rectangle = new Rectangle();
rectangle.draw(); // Output: Drawing a rectangle
rectangle.displayArea(); // Output: Area calculation is specific to each shape } }
In the Main class, objects of type Shape are created and manipulated. The abstraction allows you to treat different shapes uniformly through the Shape interface or abstract class, even though the specific implementations differ.

Abstraction helps in reducing complexity, making code more modular, and promoting a high level of code reusability. It allows you to work with objects at a higher level of abstraction, focusing on what an object does rather than how it achieves its functionality.
