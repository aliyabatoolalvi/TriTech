Inheritance

In Java, inheritance is a fundamental concept of object-oriented programming (OOP) that allows a class (subclass or derived class) to inherit properties and behaviors from another class (superclass or base class). This promotes code reuse and establishes a relationship between classes. Here's how inheritance is implemented;

Defining a Superclass (Base Class):

// Superclass or Base Class
public class Animal {
// Attribute
private String species;
// Constructor
public Animal(String species) {
this.species = species;
} // Method
public void makeSound() {
System.out.println("Some generic sound");
}
// Getter method
public String getSpecies() {
return species;
} }

Creating a Subclass (Derived Class):

// Subclass or Derived Class
public class Dog extends Animal {
// Additional attribute specific to Dog
private String breed;
// Constructor calling the superclass constructor using super()
public Dog(String species, String breed) {
super(species); // Call the constructor of the superclass
this.breed = breed;
}
// Override the makeSound() method from the superclass
@Override
public void makeSound() {
System.out.println("Woof! Woof!");
}
// Additional method specific to Dog
public void fetch() {
System.out.println("Fetching the ball"); } }

Using Inheritance:

public class Main {
public static void main(String[] args) {
// Creating an object of the subclass (Dog)
Dog myDog = new Dog("Canine", "Golden Retriever");
// Accessing attributes and methods of the superclass
System.out.println("Species: " + myDog.getSpecies()); // Output: Species: Canine
myDog.makeSound(); // Output: Woof! Woof!
// Accessing methods of the subclass
myDog.fetch(); // Output: Fetching the ball
} }

Key Concepts:

  • extends Keyword: In the subclass declaration, the extends keyword is used to specify the superclass from which the subclass inherits.
  • Constructor in Subclass: The subclass constructor typically calls the constructor of the superclass using super() to initialize the inherited attributes.
  • Method Overriding: If a method in the subclass has the same signature (name, return type, and parameters) as a method in the superclass, it is said to override the superclass method. The @Override annotation is used to indicate that a method is intended to override a superclass method.
  • Access Modifiers: Inherited members (fields and methods) retain their access modifiers. If a member is declared as public or protected in the superclass, it can be accessed in the subclass.


Inheritance supports the creation of a hierarchy of classes, allowing for the reuse of code and the creation of more specialized classes. It enhances code organization, maintainability, and flexibility.
