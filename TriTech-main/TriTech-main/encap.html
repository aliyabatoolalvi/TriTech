Encapsulation

Encapsulation is one of the four fundamental pillars of object-oriented programming (OOP) in Java. It involves bundling the data (attributes) and methods (functions) that operate on the data into a single unit called a class. The key idea is to hide the internal details of an object and provide a well-defined interface for interacting with it. Here's how encapsulation is achieved in Java:

Private Access Modifiers

  • The internal details of a class are often declared as private to restrict direct access from outside the class.
  • Private members (fields or methods) can only be accessed within the same class.
public class Car {
// Private attributes
private String make;
private String model;
private int year;
// Public constructor
public Car(String make, String model, int year) {
this.make = make;
this.model = model;
this.year = year;
} // Public method to access make
public String getMake() {
return make;
} // Public method to update year
public void setYear(int year) {
// Additional validation logic can be added here
this.year = year; } }

Public Getter and Setter Methods:

  • To allow controlled access to private members, public getter and setter methods are often provided.
  • Getter methods retrieve the values of private attributes, and setter methods modify the values.
public class Main {
public static void main(String[] args) {
// Creating an object of the Car class
Car myCar = new Car("Toyota", "Camry", 2022);
// Accessing private attributes using public methods
System.out.println("Make: " + myCar.getMake()); // Output: Make: Toyota
// Modifying private attribute using public method
myCar.setYear(2023);
System.out.println("Year: " + myCar.getYear()); // Output: Year: 2023 } }

By encapsulating the internal details of the Car class and providing controlled access through public methods, you ensure that the integrity of the object is maintained and that modifications to its state follow the logic defined in the class.

Encapsulation promotes data hiding, abstraction, and separation of concerns, making it easier to manage and maintain code in large-scale software development. It also enhances security by restricting direct access to sensitive information within a class.
