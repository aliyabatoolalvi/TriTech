CLasses and objects

Classes

A class is a blueprint or a template for creating objects. It defines the structure and behavior of objects. A class encapsulates data (attributes) and methods (functions) that operate on that data. Here's a simple example of a class:
// Define a class named "Car"
public class Car {
// Attributes (instance variables)
String make;
String model;
int year;
// Constructor (a special method used for initializing objects)
public Car(String make, String model, int year) {
this.make = make;
this.model = model;
this.year = year;
} // Method (function)
public void displayInfo() {
System.out.println("Car: " + year + " " + make + " " + model);
}
}

Objects

An object is an instance of a class. It is a concrete entity created from the class blueprint. Using the above Car class, you can create objects representing specific cars:
public class Main {
public static void main(String[] args) {
// Create two Car objects
Car car1 = new Car("Toyota", "Camry", 2022);
Car car2 = new Car("Honda", "Civic", 2021);
// Accessing object properties and invoking methods
car1.displayInfo(); // Output: Car: 2022 Toyota Camry
car2.displayInfo(); // Output: Car: 2021 Honda Civic
}
}
In the example above, car1 and car2 are instances of the Car class.

Key Concepts

  • Encapsulation: The bundling of data (attributes) and methods that operate on the data within a single unit (class). It helps in data hiding and abstraction.
  • Constructor: A special method used for initializing objects when they are created. It has the same name as the class and is called automatically when an object is instantiated.
  • Method: Functions defined within a class that perform actions or provide services related to the class.
  • Instance Variables: Variables that belong to an instance of a class (an object). Each object can have its own set of instance variables.
  • this Keyword: Refers to the current instance of the class. It is used to differentiate instance variables from local variables when they have the same name.

Object-oriented programming allows you to model real-world entities, relationships, and behaviors in a more natural and intuitive way, making code more modular, reusable, and easier to maintain.
